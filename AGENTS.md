# SmartSEA PPA Demo Agent Notes

Goal: keep this repo focused on selling SmartSEA as the passenger experience intelligence layer for PPA.

Rules:
- Do not base copy on prior engagement, internal context, or insider access.
- Position screens and CMS as channels; position SmartSEA as the real-time data, journey logic, and passenger guidance layer.
- Keep raw tender, email, proposal, and source-material files local in `SUPPORTING_SOURCE_MATERIAL_ORIGINALS/`; never track them.
- Prefer concise reusable automation over repeated manual checking.
- Before commits, run the smallest relevant validation first: `npm run validate`, then `npm run build` for app changes.
- Commit and push each meaningful milestone independently.

Core message:
- PPA can procure screens, cabling, and CMS separately.
- Passengers do not lack signs; they lack clarity on what to do next.
- SmartSEA upgrades the experience by feeding any capable CMS with live, context-aware instructions.
